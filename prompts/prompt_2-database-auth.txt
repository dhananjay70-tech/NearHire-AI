PART 5: AUTHENTICATION ENDPOINTS
==================================================

Mount auth routes at:

/api/auth

--------------------------------------------
POST /api/auth/register
--------------------------------------------

Request body:

{
  "name": "Neha Kumari",
  "email": "neha@example.com",
  "password": "Password@123"
}

Validation:

- name must contain at least 2 characters
- email must be valid
- password must contain at least 8 characters
- password must contain uppercase
- password must contain lowercase
- password must contain a number
- password must contain a special character

Behaviour:

- Trim name.
- Convert email to lowercase.
- Return 409 if email already exists.
- Hash password using bcrypt with an appropriate salt round.
- Create user with role USER.
- Automatically create a user_preferences row.
- Use a database transaction so user and preferences are created together.
- Generate JWT.
- Store JWT in an httpOnly cookie.
- Return status 201.
- Never return password_hash.

--------------------------------------------
POST /api/auth/login
--------------------------------------------

Request:

{
  "email": "neha@example.com",
  "password": "Password@123"
}

Behaviour:

- Normalize email.
- Return the same "Invalid email or password" response for an unknown email or wrong password.
- Return 401 for invalid credentials.
- Return 403 for inactive users.
- Compare password using bcrypt.
- Generate JWT.
- Store JWT in an httpOnly cookie.
- Return safe user data.

--------------------------------------------
GET /api/auth/me
--------------------------------------------

Protected route.

Behaviour:

- Read authentication token.
- Fetch current user and preferences.
- Return 401 for missing, expired or invalid token.
- Never return password_hash.

--------------------------------------------
POST /api/auth/logout
--------------------------------------------

- Clear the nearhire_token cookie.
- Return a success message.

--------------------------------------------
PATCH /api/auth/profile
--------------------------------------------

Protected route.

Allow only:

- name
- phone
- profile_image_url
- address
- city
- state
- postal_code
- latitude
- longitude

Do not allow:

- email
- password
- password_hash
- role
- is_active

--------------------------------------------
GET /api/auth/preferences
--------------------------------------------

Protected route.

Return current user preferences.

--------------------------------------------
PATCH /api/auth/preferences
--------------------------------------------

Protected route.

Allow:

- preferred_role
- minimum_salary
- maximum_distance_km
- preferred_job_types
- preferred_work_modes
- preferred_locations

Validate:

- minimum_salary cannot be negative
- maximum_distance_km must be greater than 0
- JSON fields must be arrays

PART 6: JWT AND COOKIE
==================================================

JWT payload must contain only:

{
  "userId": "...",
  "role": "USER"
}

Read:

- JWT_SECRET
- JWT_EXPIRES_IN

Cookie:

- name: nearhire_token
- httpOnly: true
- sameSite: lax
- secure: true only in production
- maxAge: 7 days for the current development configuration
- path: /

Authentication middleware must support:

1. nearhire_token cookie
2. Authorization: Bearer <token> as fallback

Cookie token has priority.

Attach this to the request:

req.user = {
  userId,
  role
}
